# Software-Testing-Automation-Portfolio
Portfolio artifacts from CS‑320, including Contact Service implementation and full JUnit test suites. Demonstrates software testing, validation, automation, and quality assurance through modular Java design and requirement‑driven test coverage.

# How can I ensure that my code, program, or software is functional and secure?
I ensure my code is functional and secure by using structured unit testing and strict input validation. Writing JUnit tests for every requirement allowed me to verify that each method behaved correctly, handled edge cases, and rejected invalid data. This approach helped me catch logic errors early and maintain confidence that the Contact and Task services were reliable and safe to use.

# How do I interpret user needs and incorporate them into a program?
I interpret user needs by breaking requirements into clear, testable behaviors. For both projects, I translated each requirement, such as field length limits, unique IDs, or update rules, into specific tests. This ensured that the final program matched what the user actually needed rather than what I assumed they wanted.

# How do I approach designing software? 
I approach software design by focusing on modularity and clarity. Creating separate classes for data objects and service logic made the code easier to test, maintain, and extend. Test‑driven development guided my design decisions, helping me build software that is organized, predictable, and aligned with real‑world quality assurance practices.
